# Reflection on Sprint-1

## Learning points
- Customer review: Presenting a live demo rather than screenshots led to more specific and actionable feedback
- MVP-v1 delivery: all team meetings must be documented. Lack of documentation betweeen meetings led to fixing data format in fronted part of some endpoints in last minute before demo

## Validated assumptions
- **Confirmed:** customer changed priority of sprint-2 from event parsing only to /game page finalizing (important), event parsing (optional)
- **Rejected:** we planned to implement file uploading function to upload game media, but customer decided to keep links
- **Confirmed:** customer decided to keep visual style from mvp-v0

## Friction and gaps
- Documentation filling duties were postponed untill last days, because docs required functional mvp and customer meeting
- Project containers several times went down due to lack of DevOps knowlegde
- Meetings without all team members generated several knowledge gaps, which lead to several bugs and API misscomunications

## Planned response

- Team will search to container testing tool to improve overall deploy stability
- Documentaional work will be splitted in parts, which are not require full preparations from project's
- All meetings will be documented and notes will be published on Miro board
